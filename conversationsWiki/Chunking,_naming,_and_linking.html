<h1>Chunking, naming, and linking</h1> <p>Pete likes to call this "Chunking, naming, linking", in that order. <em>(Why, Pete?)</em></p> <h2>2021-09-05 "chunking, naming, and linking" notions.</h2> <p>This page contains some preliminary thinking by <a class="wikilink" href="/developer-massive-wiki/conversationsWiki/People/Bill_Anderson.html">Bill Anderson</a>.</p> <ol> <li> <p>The ideas of "chunking, naming, and linking" are related to managing wiki contents, as well as information system architecture, design, and deployment. I have questions about each of these notions. These questions are also about <span class="incipient-wikilink">classification</span>.</p> </li> <li> <p>First,"chunking": this is a term used in <span class="incipient-wikilink">cog-sci</span> to name the human mental work of putting ideas or text entities together under one category or in one place. Sometimes chunking requires breaking large items into smaller pieces; e.g., a large text document into separated sections, such as paragraphs, or even sentences or phrases.</p> <ul> <li>are all chunking schemes hierarchical? (Need to explore <a class="wikilink" href="/developer-massive-wiki/conversationsWiki/Faceted_classification.html">Faceted classification</a>)</li> <li>is tagging chunking? (Bill's friend <a href="http://alum.mit.edu/www/martymcgowan">Marty McGowan</a> asserts that tags either name a collection or a property of an information item.)</li> </ul> </li> <li> <p>Second, "naming": in computer-based information systems every item needs a name. The computer system needs ways of representing the digital items so that they can be accessed and maintained over time. And in our lived experience we use names to enable sharing our experiences of the world with each other. - all names are arbitrary and socially determined - naming schemes? whatever they are they need to be agreed upon to support long-lived linking.</p> </li> <li> <p>Third, "linking": in my mind this notion refers to practices of connecting separate information items by explicit relations (and relationships?).</p> </li> </ol> <ul> <li> <p>Zettelkasten: this <a href="https://zettelkasten.de/introduction/">introduction</a> provides a generative description of the method.</p> </li> <li> <p>Unix OS file system: how files are managed in the directory system; how hierarchy of the file system is related to maintenance of file system integrity. (cf., <a href="https://bandstands.praxis101.net/references/1978_ritchie_unixtimesharingsystem">Bell System Technical Journal: The UNIX Operating System, July 1978</a>)</p> </li> <li> <p>linking on wikis and elsewhere: in order to link an item it needs a name, hence item 3, above. In the world-wide-web lands, links are represented by <a class="wikilink" href="/developer-massive-wiki/conversationsWiki/URIs_URLs_and_URNs.html">URIs URLs and URNs</a>.</p> </li> </ul> <h2>2021-09-17 "wiki thoughts"</h2> <p>Following up on some thoughts from <a class="wikilink" href="/developer-massive-wiki/conversationsWiki/People/Peter_Kaminski.html">Peter Kaminski</a>.</p> <ol> <li> <p>Information organization on a wiki is focused on linking and chunking rather than classification. Change that.</p> </li> <li> <p>Put ideas into a wiki page and create links for the associations the arise during the writing.</p> <ul> <li>(Speculation) This method will lead to a large number of linked pages, any of which could be turned into a primary essay. (One example of <a class="wikilink" href="/developer-massive-wiki/conversationsWiki/People/Bill_Anderson.html">Bill Anderson</a>'s notion of picking up a connected graph by one of the nodes and shaking to produce a hierarchy.)</li> </ul> </li> </ol>
