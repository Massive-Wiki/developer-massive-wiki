These include:</p> <ul> <li> <p>Footnotes (how the heck do you specify these in Obsidian?)</p> </li> <li> <p>Strikethrough</p> <ul> <li>like <del>this text has been struck through</del></li> <li>but this has not</li> </ul> </li> <li> <p>Fenced code:</p> </li> </ul> <pre><code class="language-shell">$ ls -lrt $ date -u </code></pre> <ul> <li> <p>and, you know, wikilinks; e.g.,</p> <ul> <li> <p><a class="wikilink" href="/developer-massive-wiki/Gemstone_Names.html">Gemstone Names</a></p> </li> <li> <p><a class="wikilink" href="/developer-massive-wiki/Wiki_Theory_and_Culture/WikiNow.html">WikiNow</a></p> </li> <li> <p><a class="wikilink" href="/developer-massive-wiki/Link_workbench/testdir/text_only_wiki_page.txt">text format page</a></p> </li> <li> <p>images will need some special handling</p> </li> <li> <p>here is an image from the MassiveWiki <code>_attachments</code> folder in CommonMark format:<br /> <code>![alternate text](link_to_image_file)</code>:</p> </li> </ul> <p><img src="_attachments/2021-11-11-Milosz.jpeg" alt="The purpose of poetry" /></p> <ul> <li>and here is an image from a different directory using Obsidian wikilink format: <code>![[image_file_name]]</code>:</li> </ul> <p><img src="/developer-massive-wiki/_attachments/refactoring-massive-wiki-wikis-2021-07-29-002.png" alt="an image with no alt text" /></p> </li> <li> <p>and for good measure another image in yet another directory using Obsidian wikilink format: <code>![[image_file_name]]</code>: <img src="/developer-massive-wiki/conversationsWiki/images/Anderson_2017_semanticWebEntities.png" alt="an image with no alt text" /></p> </li> </ul> <p>Other tests of using Mistletoe to render Markdown to HTML for Massive Wikis are needed.
