<h1>How to Clone with Git</h1> <p>Use case: We want to sync a new repo / vault down from GitHub to my local computer.</p> <p>There are several methods/tools you can use. Choose one that suits you best.</p> <h2>0. Don't clone it to your local machine at all</h2> <p>You can read and edit (if you have permissions) the wiki through the GitHub web interface. It's very clunky, but it can be doe.</p> <h2>1. command line</h2> <pre><code class="language-shell">cd ~/Documents/GitHub git clone https:// ... </code></pre> <pre><code>- easiest, but sort of inscrutable unless you're familiar with command line </code></pre> <h2>2. Obsidian Git command in Obsidian</h2> <p>Use the Command Palette in Obsidian and search for "clone".</p> <p>No other tool required, but the user interface and workflow is a little clunky.</p> <h2>3. Use your IDE (such as Pulsar Editor)</h2> <p>If you're using an IDE for other things, you can also use its Git integration for wikis.</p> <p>Only good if you're familiar with your IDE, and find it easier than using Obsidian Git.</p> <h2>4. GitHub Desktop</h2> <p>GitHub Desktop is a free and open source Git UI client. It also has good authentication integration with GitHub.</p> <h2>Other Git UI Clients</h2> <p>You may find other Git UI clients useful. See <a class="wikilink" href="/developer-massive-wiki/Git_GUI_Clients.html">Git GUI Clients</a>.</p>
