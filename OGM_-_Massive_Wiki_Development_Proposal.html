<h1>OGM - Massive Wiki Development Proposal</h1> <p>We are excited to present a comprehensive proposal for the development and enhancement of features for Massive Wiki Builder, part of the Massive Wiki ecosystem. Open Global Mind, spearheaded by Jerry Michalski, is the project sponsor. The features proposed here directly benefit OGM's "NeoBooks" project, along with other OGM efforts.</p> <p>Our project lead, Peter Kaminski, along with our skilled development team consisting of Pete, Bill Anderson, and John Abbe, are committed to delivering high-quality outputs tailored to OGM's needs. Where necessary, we will engage specialized contractors as needed (particularly for HTML/CSS theme development).</p> <p>Our proposal encompasses a suite of features designed to enhance user experience and increase functionality across the Massive Wiki platform.</p> <p>Individual features are listed below.</p> <h2>Transclusion, $2000</h2> <p>MWB acts similar to Obsidian when it sees <code>![[Other page]]</code> in the Markdown -- the "Other page" is included verbatim in the transcluding page.</p> <p>Timeline:</p> <ul> <li>Feature implemented in MWB v3.2.0, 2024-03-27, still to do: <ul> <li>minimal documentation, to be completed by 2024-04-15</li> <li>deployment to <code>lionsberg.wiki</code> and <code>wiki.openglobalmind.com</code> by 2024-04-30</li> </ul> </li> </ul> <p>Definition of done:</p> <ul> <li>feature is built into a version of MWB</li> <li>feature is deployed to <code>lionsberg.wiki</code> and <code>wiki.openglobalmind.com</code></li> <li>MWB still operates as before (doesn't crash, doesn't need extra arguments)</li> <li>feature has been tested</li> <li>feature has been minimally documented in MWB's README.md</li> <li>don't get caught in an infinite loop with self-transclusion</li> <li>page has a small icon that is a link to the transcluded page</li> </ul> <p>Future enhancements:</p> <ul> <li>some way of enabling / disabling a header that gives the filename / page title</li> <li>some way of enabling / disabling an H1 header that's the first Markdown line</li> <li>suppresses YAML frontmattter</li> </ul> <hr /> <h2>Finish PyPI module publication</h2> <p>Provide Massive Wiki publishing as a PyPI(Python Package Index) module that enables installation using the Python <code>pip</code> capability; e.g., if module name is "MWB-publish":</p> <pre><code class="language-shell">pip install MWB-publish </code></pre> <p>Timeline:</p> <ul> <li>PyPI module released (in the wild), 2024-06-19: still to do: <ul> <li>implementation of module <code>init</code> and <code>publish</code> features</li> <li>PyPI documentation (there are standards)</li> <li>decide on PyPI module name (thinking web publishing;-- more than Massive Wiki publishing)</li> <li>module uses <code>pytest</code></li> </ul> </li> </ul> <p>Definition of done:</p> <ul> <li>use Poetry checklist at the bottom of this chat: https://chat.openai.com/share/011ae8a7-c61e-4c3e-82ad-83b2b517e739</li> </ul> <ul> <li>module available on the PyPI server</li> <li>feature is deployed to <code>massive.wiki</code> and <code>developer.massive.wiki</code> (others?)</li> <li>MW publish operates as MWB v3 does for publishing existing Massive Wikis</li> <li>module has a working and documented <code>pytest</code></li> <li>Massive Wiki documentation reflects use of PyPI module</li> </ul> <p>Future enhancements:</p> <ul> <li>n/a</li> </ul> <hr /> <h2>Edit this page, phase 1 $600</h2> <p>Add an "edit this page" button to every Massive Wiki page that allows a wiki reader to edit the page content and stage the changes for incorporation into the wiki.</p> <p>Timeline:</p> <ul> <li>Feature implemented in MWB v3.3.0, 2024-05-15, still to do: <ul> <li>implementation</li> <li>documentation, perhaps with a short video demo?</li> <li>deployment to <code>massive.wiki</code> and <code>wiki.openglobalmind.com</code> and <code>developer.massive.wiki</code>?</li> </ul> </li> </ul> <p>Definition of done:</p> <ul> <li>feature is built into a version of MWB</li> <li>feature is deployed to <code>massive.wiki</code> and <code>wiki.openglobalmind.com</code></li> <li>feature has been tested (by MW users and well as developers)</li> <li>feature has been documented in MWB's README.md <ul> <li>documentation includes a "how-to-use" this feature</li> </ul> </li> </ul> <p>Future enhancements:</p> <ul> <li>n/a</li> </ul> <hr /> <h2>2-3 New Themes</h2> <p>Create 2 or 3 new themes that are clean and attractive, and have affordances for Sidebar nav. Themes must work well at desktop and mobile aspect ratios, including nav (e.g., collapse nav to a hamburger menu).</p> <hr /> <h2>Draft Project Proposal Items</h2> <ul> <li>Edit This Page button, phase 1, $600</li> <li>Transclusion, ~$2000 <ul> <li>option not yet included in proposal: render a permalink to transcluded pages</li> </ul> </li> <li>finish PyPI module publication (update mechanism), $600</li> <li>2-3 new themes, $1000</li> <li>multiple page templates, $800 <ul> <li>multiple sidebars?</li> </ul> </li> <li>embed PDFs, $400</li> <li>improve documentation, $200 <ul> <li>document favicon setup</li> <li>document how to change theme colors, etc.</li> <li>document upgrade process</li> <li>other useful docs</li> </ul> </li> <li>Discourse-based commenting system investigation / architecture, $400</li> </ul> <p>Current rough budget 600 + 2000 + 600 + 1000 + 800 + 400 + 200 + 400 = 6000 (grant will be &lt;= 6000)</p>
