<h1>2022-10-24 some markdown testing</h1> <p>This is an Obsidian note to observe what happens in MWB when making bulleted (un-ordered) lists.</p> <ul> <li>like this bullet item here</li> <li>and this other one</li> <li>and yet a third one following on</li> <li>without adding any spaces at the</li> <li>end of the lines; just CRs</li> <li>however, a blank line is needed before starting the list</li> </ul> <h2>later that same day (2022-10-24)</h2> <ul> <li>we add this page to developer wiki</li> <li>just to keep ourselves honest about what MWB does with un-ordered lists</li> <li>so this list only has CRs at the end of the lines</li> </ul> <h3>but this list here</h3> <ul> <li>has explicit two space characters</li> <li>at the end of each line</li> <li>which seems like a lot to ask</li> <li>of the hapless folk just trying to</li> <li>make a list.</li> </ul> <h4>notes on what was observed</h4> <ul> <li>a line of plain text followed by items with leading hyphens "-" are formatted by MWB as an HTML paragraph, not as an unordered list.</li> <li>one solution is to place a blank line between the sentence and the listed items. this is formatted as an un-ordered list.</li> <li>last observation: any kind of header can be followed by an un-ordered list, and list items can be terminated by CRs (i.e., no blank spaces are needed at the end of the list items)</li> </ul>
