<h1>2023-01-20 MWB Program flow</h1> <p>This note outlines the steps taken to build a Massive Wiki static website. The code outlined is <code>mwb.py</code>, version 2.1.0.</p> <p>The <code>mwb.py</code> <code>main</code> function steps:</p> <ul> <li> <p>initialize: parse command line arguments</p> </li> <li> <p>store configuration</p> </li> <li> <p>get Jinja2 env</p> </li> <li> <p>set up lunr filename and sitepath indices</p> </li> <li> <p>render the wiki:</p> <ul> <li> <p>remove and recreate output directory</p> </li> <li> <p>generate dict of filenames and wikipaths to support wikilinks (first loop through all files)</p> </li> <li> <p>copy wiki to output; render .md files as HTML</p> <ul> <li>second loop through files <ul> <li>parse Markdown files (frontmatter and markdown body)</li> <li>collect git commit messages and times</li> <li>append to all_pages list</li> </ul> </li> </ul> </li> <li> <p>build lunr search index (if specified)</p> </li> <li> <p>set up search web page</p> </li> <li> <p>build all_pages web page</p> </li> </ul> </li> </ul> <hr /> <p>cf. <a class="wikilink" href="/developer-massive-wiki/_attachments/MWB-dataflowV3.png">MWB-dataflowV3.png</a></p> <hr /> <h2>Pete's notes, 2023-02-17</h2> <h3>Paths for all non-hidden Markdown files</h3> <ul> <li>absolute </ul> <h3>Files</h3> <p>All non-hidden files of any type:</p> <ul> <li>are copied from wiki to output</li> </ul> <p>All non-hidden Markdown files:</p> <ul> <li>name of file is used as a key in the list of wikilink targets</li> <li>are entered into the search index</li> <li>file is parsed and <ul> <li>a <code>.html</code> version is saved to the location that will be next to the <code>.md</code> file</li> <li>a <code>.json</code> extract of the YAML frontmatter is saved to the location that will be next to the <code>.md</code> file</li> </ul> </li> </ul> <h3>Wikilinks</h3> <p>If we accumulate all links seen during Markdown parsing, after all Markdown files are parsed, we can compare that to the list of wikilink targets, and determine:</p> <ul> <li>unlinked files (could write an HTML report file)</li> <li>links to non-existent files (each of which could be created as a stub HTML file)</li> </ul> <p>These lists could also be output as log messages.</p> <p><a class="wikilink"
