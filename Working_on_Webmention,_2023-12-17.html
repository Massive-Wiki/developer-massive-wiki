<h1>Working on Webmention, 2023-12-17</h1> <div class="h-entry"> <h1 class="p-name">Working on Webmention, 2023-12-17</h1> <a class="u-url" href="https://developer.massive.wiki/working_on_webmention,_2023-12-17">permalink</a> (perhaps not needed) <div class="e-content"> I (Pete) am working on Webmention as part of IndieWebCamp San Diego 2023. See <https://etherpad.indieweb.org/IndieWebCamp_San_Diego>. <p><a href="https://ragt.ag/">Angelo</a> is graciously helping me by answering newbie questions. :-)</p> <p>My first experiment is to use a service to send Webmentions; the services I've looked at will send a Webmention for links inside an <code>h-entry</code>.</p> <p>The current plan is to wrap the whole body of wiki pages in <code>h-entry</code>; this page will be doing that.</p> </div> </div> <p>The content above works! We might not need the <code>u-url</code> permalink (for future testing). It was parsed and a webmention was sent by <a href="https://telegraph.p3k.io/">https://telegraph.p3k.io/</a>, with Pete manually clicking the send button.</p> <p>Next up: automate sending webmentions when the website is built.</p> <p>Pseudocode:</p> <ul> <li>for each wiki page <ul> <li>has it changed since we last checked it for sending webmentions? <ul> <li>if so, for each outgoing link on the page <ul> <li>send a webmention from this page's url to the url of the outgoing link</li> <li>update the "last checked for sending webmentions" time for this page</li> </ul> </li> </ul> </li> </ul> </li> </ul> <p>See <a href="https://indieweb.org/Webmention-developer#Libraries">https://indieweb.org/Webmention-developer#Libraries</a> for libraries.</p> <p>Consider doing an RSS feed at the same time.</p>
