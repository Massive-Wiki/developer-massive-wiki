<h1>How to set up and use a Massive Wiki</h1> <p>2024-03-15: some notes to scope the work needed to put a Massive Wiki into practice</p> <hr /> <p>What are the pre-requisites for using Massive Wiki?</p> <ol> <li>Familiar with Markdown formatting of text files</li> <li>Some experience using <code>git</code> and <a href="https://github.com">Github</a></li> </ol> <hr /> <p>What are the basic steps to getting a Massive Wiki working?</p> <ol> <li>Gather the Markdown files to be worked on and shared into a folder (this folder can contain subfolders)</li> <li>Establish this folder as a <code>git</code> repository that is hosted on Github (use <code>massive-wiki-publishing-kit</code> Github repository template?)</li> <li>Share this repository with collaborators and establish practices to support sharing the document work (writing, reviewing, editing, etc.) (You probably already know how to work together. Using <code>git</code> is best with some rules of practice.)</li> </ol> <hr /> <p>What are the steps needed to publish the documents on the Web?</p> <ol> <li>Install and configure Massive Wiki Publishing Kit <ul> <li>install web theme <code>basso</code></li> <li>set web site configuration properties in <code>mwb.yaml</code></li> </ul> </li> <li>Set up an account on a web-hosting service such as <a href="https://netlify.app">Netlify</a></li> <li>Configure Netlify to rebuild your website when the Github repository content is updated.</li> </ol>
