<h1>Start a Massive Wiki - how-to</h1> <p>A Massive Wiki supports the sharing of versions of files in Markdown format. Markdown, Shared, Versioned, Files led to an acronym, MaSVF, that is pronounced like the word "Massive". So writing shared, versioned, Markdown files makes it "Massive";-- what makes it a "Wiki" is creating links to other pages while you write your thoughts in this one, a wiki property referred to as <a class="wikilink" href="/developer-massive-wiki/Wiki_Theory_and_Culture/Link_As_You_Think.html">Link As You Think</a>. A Massive Wiki is a collection of files in markdown format that contain links to each other and to other resources both local and on the internet.</p> <p>Establishing and using a Massive Wiki starts with answering a few questions.</p> <p><strong>First</strong>: what software apps are used to write and edit Markdown Files? One app that supports Markdown and inter-file linking is <a href="https://obsidian.md">Obsidian</a>. There are other well-supported apps for creating wiki-linked notes, such as <a href="https://logseq.com">Logseq</a>. Plain text editors such as Emacs, Vim, Notepad, (more??) can be used but wiki-linking is often left without support from the app.</p> <ul> <li>TODO: clarify this list and include links</li> </ul> <p><strong>Second</strong>: how are wiki files shared and versioned? One way to do this is to use a Git-based system, such as <a href="https://github.com">GIthub</a>, <a href="https://gitlab.com">Gitlab</a>, or <a href="https://codeberg.org">Codeberg</a>. Our current primary wiki practice uses Github. Git services manage the versioning of wiki files well; sharing practices need to be established for each wiki. We have <a class="wikilink" href="/developer-massive-wiki/File_Sharing_and_Versioning.html">suggestions</a>.</p> <p>With these two questions answered a Massive Wiki can be created and put into practice. One further question remains.</p> <p><strong>Third</strong>: will this wiki be published to the World-Wide Web? The Massive Wiki team provides a publication method, Massive Wiki Builder (MWB), that renders the content of the wiki as a static web site. For a Github wiki repository, MWB publishing can be enabled by Github, or by another hosting service such as <a href="https://netlify.app">Netlify</a>.</p>
