<h1>Installing Git and GitHub CLI on Mac</h1> <p><em>Instructions section moved to <a href="https://prosefusion.cloud/how-tos/installing_git_and_github_cli_on_mac">Installing Git and GitHub CLI on Mac</a> (Prose Fusion wiki).</em></p> <h2>PK, 2023-09-09</h2> <p>I'm exploring the installation of Git on a fresh Mac. I'm using VirtualBuddy and macOS 12.6.1.</p> <p>Notes are below, but the result is the easiest path for non-technical people is to install Homebrew, then GitHub CLI, then use GitHub CLI to authenticate to GitHub.</p> <h3>Via Developer Tools</h3> <p>In Terminal, type <code>git --version</code>.</p> <p>You'll get a popup:</p> <blockquote> <p>The "git" command requires the command line developer tools. Would you like to install the tools now?</p> </blockquote> <p>Click "Install", and wait until it's done.</p> <p>In Terminal, type <code>git --version</code>. You'll get an answer:</p> <p><code>git version 2.37.1 (Apple Git-137.1)</code></p> <p>(Note, on my machine, the version I normally use is not Apple's, and it is version 2.38.1.)</p> <h3>Via Homebrew and Command Line</h3> <p>Worked great.</p> <p>Go <a href="https://brew.sh/">https://brew.sh/</a>, copy the command line under the "Install Homebrew" header, paste into Terminal, follow prompts. (Asks for confirmation via RETURN, needs account password a couple of times, prompts you to set up environment for your shell.)</p> <p>After setup, <code>git --version</code> returns:</p> <p><code>git version 2.37.1 (Apple Git-137.1)</code></p> <p>(Under the hood, the Homebrew installer uses <code>xcode-select--install</code> or similar to get Command Line Tools installed.)</p> <h3>Via Homebrew <code>.pkg</code></h3> <p>On running package, the first thing that pops up is a dialog:</p> <blockquote> <p>Command Line Tools (CLT) are missing You must install the Command Line Tools (CLT) before installing Homebrew by running 'xcode-select - - install' from a Terminal.</p> </blockquote> <p>I stopped there, and recommend the Command Line install above.</p> <h3>To <code>git clone</code> a repo from GitHub</h3> <p>The green <code>Code</code> button on a repo's home page offers <code>https:</code> or <code>GitHub CLI</code> options. It no longer offers a <code>git:</code> option.</p> <h3>Installing <code>gh</code></h3> <p>Navigate to <a href="https://cli.github.com">https://cli.github.com</a>. On that page, you have a choice:</p> <blockquote> <p>brew install gh or Download for Mac</p> </blockquote> <p>The "Download for Mac" option downloads a zip file, with a .bin file and a man page. I do not immediately see instructions on where to put the .bin file. Putting it in the general <code>Applications</code> folder does not set it up to run in Terminal.</p> <p>The obvious place is <code>/usr/local/bin</code>, and that directory is in the default zsh path; however <code>/usr/local/bin</code> does not yet exist.</p> <p>Ugh.</p> <p>I am tempted to think that installing Homebrew and then installing <code>gh</code> with <code>brew install gh</code>, while adding <em>yet another</em> package, may be more straightforward and less error-prone for non-technical users than the "Download for Mac" option.</p>
