<h1>Starter wiki learnings (Bill)</h1> <p>2022-07-29</p> <p>Reflections on using the massive-wiki-starter as the template for a personal Massive Wiki.</p> <ul> <li>Massive Wiki pre-requisites: <ul> <li>make decisions on (1) authoring, (2) sharing, (3) versioning, and (4) having a public web site of wiki content.</li> <li>(1) authoring: Obsidian (or other app) to create Markdown files</li> <li>(2) sharing: Github (or other Git repository hosting) is one choice</li> <li>(3) versioning (is provided by Git repositories)</li> <li>(4) public web site requires web hosting (netlify.app is one option)</li> </ul> </li> <li>Decisions made; let's get started: <ul> <li> <p>choose a folder on your computer to contain your wiki content (Bill uses ~/Documents/myWikis/ as the base and create a different folder for each wiki)</p> </li> <li> <p>we are using Github, so this is one way to proceed:</p> </li> <li> <p>(1) general outline: use massive-wiki-starter repository code to populate the new wiki-repository (download .zip file? fork the repo?); initialize this content into its own Github repository; configure and add the MWB and MWT submodules (this is getting complicated); make sure everything is up-to-date; push the current state as a baseline version of this new wiki(?)</p> </li> <li> <p>(2022-07-31 update: https://cli.github.com/manual has some useful examples of how to proceed with a starter-wiki repository:<br /> e.g., <code>$ gh repo create mynewrepo --public --source=. --remote=upstream</code> also see <a class="wikilink" href="/developer-massive-wiki/Starter_wiki_workbench/Github_-_getting_started_notes.html">Github - getting started notes</a></p> </li> <li> <p>(2) edit the README.md file with information you want on your wiki home-page</p> <ul> <li>since we said "yes, we want a public website" now is the time to edit the MassiveWikiBuilder configuration file <code>.massivewikibuilder/mwb.yaml</code> , cf. <span class="incipient-wikilink">MWB configuration</span></li> <li>using a Sidebar? now is the time to add content to <a class="wikilink" href="/developer-massive-wiki/Sidebar.html">Sidebar</a></li> <li>since we are implementing a Sidebar now is the time to edit <code>wiki_dir/netlify.toml</code> and insure that the mwb command is using the basso theme</li> </ul> </li> </ul> </li> </ul>
