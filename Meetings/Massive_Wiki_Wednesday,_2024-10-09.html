<h1>Massive Wiki Wednesday, 2024-10-09</h1> <h2>Topics</h2> <ul> <li>Guard(s) for nxc</li> <li>Working towards the pypi release</li> </ul> <h2>Guard(s) for nxc</h2> <ul> <li>If the user starts <code>nxc build</code> but there isn't an <code>.nxc</code> directory, stop and say "no .nxc directory found, you need to run <code>nxc init</code>, see instructions for more information"</li> <li>2024-10-10: guard implemented; preliminary tests pass; deployed to test-pypi as version 0.1.29</li> </ul> <h2>Working towards the pypi release</h2> <h3>Gotta Have</h3> <ul> <li>Name</li> <li>Define terminology for our model of operation <ul> <li>documents</li> <li>markdown</li> <li>directory / folder</li> <li>repo / vault</li> <li>wiki</li> <li>static site, static site builder</li> </ul> </li> </ul> <h3>Nice to Have</h3> <ul> <li>Graceful fallbacks without README.md or index.md or whatever</li> </ul> <hr /> <h2>Graceful fallbacks without README.md or index.md or whatever</h2> <p>Observation: We shouldn't depend on the user knowing or caring about the convention of README.md -&gt; index.html.</p> <p>Therefore:</p> <ul> <li>If there's README.md, use it as index.html</li> <li>If there's no README.md, but something else like index.md, INDEX.md, index.html, use that as index.html (possibly converting .md to .html)</li> <li>If there's nothing like an index/README file, create a simple and useful index.html, and log an INFO message to that effect. The index.html should recommend to the user to create a README.md or Index.md file.</li> </ul>
