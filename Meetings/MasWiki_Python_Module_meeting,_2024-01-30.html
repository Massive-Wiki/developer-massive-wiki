<h1>MasWiki Python Module meeting, 2024-01-30</h1> <h2>Off Topic</h2> <ul> <li>https://memgpt.ai/</li> <li>https://pi.ai/ - a good conversationalist, free to start</li> </ul> <h2>Use Cases, Personas</h2> <ol> <li>I'm installing MasWiki because I want to turn Obsidian vaults into websites -&gt; install MasWiki <ul> <li>"website" - does that mean local? cloud? both?</li> <li>"install MasWiki" -- globally, or with a venv</li> </ul> </li> <li>I want a quick and easy website (and a static site is okay) -&gt; install Publishing Toolkit Further notes on this topic: <a class="wikilink" href="/developer-massive-wiki/web_and_wiki_publishing_-_scenarios_and_workflows.html">web and wiki publishing - scenarios and workflows</a></li> </ol> <h2>Package Name</h2> <ul> <li>temporary name is MasWiki</li> <li>future name TBD, needs to be friendly for people who want a static site, but not necessarily a wiki</li> </ul> <h2>Research: "init" methods used by other tools</h2> <ul> <li>11ty</li> <li>Hugo</li> <li>Express.js?</li> <li>MemGPT: "quickstart" command, Set the base config file with a single command</li> </ul> <h2>General Architecture</h2> <h3>MasWiki -- the builder</h3> <ul> <li>distributed as a PyPI module</li> <li>has an "init" function to get somebody going <ul> <li>includes mwb.yaml, a simple theme, and package.json</li> </ul> </li> </ul> <h3>Publishing Toolkit - template repo to get a static website up easily</h3> <ul> <li>netlify.toml</li> <li>.massivewikibuilder directory <ul> <li>requirements.txt is used by Netlify to install MasWiki (replaces git submodule)</li> <li>instructions to install MasWiki module locally, but installing MasWiki for Netlify is handled automatically</li> </ul> </li> </ul> <h2>MasWiki Actual Getting Started Instructions</h2> <ul> <li>create files like <span class="incipient-wikilink">PK MasWiki Getting Started</span> or <span class="incipient-wikilink">BA MasWiki Getting Started</span></li> </ul> <p><em>stuff goes here</em></p> <h3>Installed Globally, used for multiple wikis</h3> <ul> <li>skip venv, because it's global anyway</li> <li>typical uses option A: <ul> <li><code>maswiki --create ~/Documents/GitHub/new-wiki</code></li> <li><code>maswiki --build ~/Documents/GitHub/existing-wiki</code></li> <li><code>maswiki --build .</code> # looks for <code>.maswiki/maswiki.yaml</code> ?</li> </ul> </li> <li>typical uses option B (probably the more appropriate way): <ul> <li><code>mkdir new-wiki; cd new-wiki; maswiki init; emacs .maswiki/maswiki.yaml</code></li> </ul> </li> </ul> <h2>Publishing Toolkit Actual Getting Started Instructions</h2> <h3>User that doesn't know or care how it works, just needs a website</h3> <ul> <li>use Publishing Toolkit repo as a template to create a new wiki</li> </ul>
