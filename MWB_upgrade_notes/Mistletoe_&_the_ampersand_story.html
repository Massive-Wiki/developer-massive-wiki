<h1>Mistletoe &amp; the ampersand story</h1> <p>2023-05-10:<br /> This is a test page to verify, and, if needed, validate a fix for Massive Wiki Builder issue number 49: <a href="https://github.com/peterkaminski/massivewikibuilder/issues/49"># Ampersand in a wiki link on the Sidebar leads to 404 on website</a></p> <p>Let's see if this same issue shows up here on developer-massive-wiki. And it does.</p> <p>Next steps: 2023-05-11: leaving the mistletoe wikilink cargo cult:</p> <ol> <li> <p>in the Obsidian Markdown page a wikilink has these parts contained within double square brackets and separated by a vertical bar (<code>|</code>):</p> <ul> <li>name of the wiki page</li> <li>optional display text</li> </ul> </li> <li> <p>in the Mistletoe rendered HTML:</p> <ul> <li> <p><code>token.target</code> == the data in an <code>&lt;a&gt;,&lt;/a&gt;</code> tagged item; this corresponds to the optional display text (if any) in the wikilink; otherwise the page name</p> </li> <li> <p><code>self.render_inner(token)</code> == content of the <code>'href'</code> attribute in an <code>&lt;a&gt;,&lt;a/&gt;</code> tagged item; this corresponds to a weblink to the wiki page, relative to the root of the wiki directory (Obsidian vault). This value is read from a wikilinks directory constructed when running Massive Wiki Builder.</p> </li> </ul> </li> <li> <p>one way to fix this ampersand in the page name problem is to counteract the <code>html.escape()</code> use prevalent in the <code>mistletoe</code> render classes when retrieving a weblink from the <code>wikilinks</code> dictionary.</p> <ul> <li>the key line of code in <code>mistletoe_renderer/massivewiki.py</code> is this:</li> </ul> </li> </ol> <pre><code class="language-python"> wikilink_key = html.unescape(Path(self.render_inner(token)).name) </code></pre>
